PulsarNet - Network Device Management System

1. Overview

PulsarNet is a comprehensive network device management system designed for efficient backup, monitoring, and configuration of network devices. The application follows ISO/IEC 9126 standards for software quality.

2. Main Features

2.1 Device Management

  • Add, edit, and remove network devices
  • Group devices for organized management
  • Import devices from CSV files
  • Real-time device status monitoring

2.2 Backup Management

  • Manual and automated configuration backups
  • Local and remote storage options (FTP, SFTP, TFTP)
  • Flexible scheduling options (daily, weekly, custom)
  • Retention policy management
  • Support for various device connection methods (direct, jump host)
  • Multi-vendor device support with customized command sets

9. Backup Process - Detailed Overview

PulsarNet follows ISO/IEC 27001 standard practices for secure and reliable network device configuration backups. This section provides a comprehensive explanation of the backup process flow.

9.1 Backup Data Flow

The configuration backup process follows this secure path:

Target Device → Jump Host (if used) → PulsarNet Application → Storage Location

Important: Configuration data is not transferred directly from the target device to a storage server. This design ensures compatibility with enterprise environments that have strict firewall policies and network segmentation.

9.2 Backup Process Steps

  1. Connection Establishment:
    • PulsarNet connects to the target device (directly or via jump host)
    • Authentication is performed using configured credentials
    • For jump host configurations, a secure SSH tunnel is established first
  2. Command Execution:
    • Device-specific commands are executed (e.g., "show running-config" for Cisco devices)
    • Output is captured in memory by the PulsarNet application
    • No temporary files are created on jump hosts or target devices
  3. Configuration Storage:
    • Configuration is saved locally on the PulsarNet host system by default
    • Optional transfer to remote storage servers (SFTP, FTP, TFTP) if configured
    • File naming convention: [device_name]_[IP]_[timestamp].cfg
  4. Verification (Optional):
    • Configuration syntax validation
    • Comparison with previous backups for change detection
    • Backup integrity verification

9.3 Connection Types and Protocols

Connection Type Backup Protocol Use Case
Direct SSH SCP Devices with direct SSH access
Direct Telnet TFTP Legacy devices without SSH
Jump SSH → Device SSH SFTP Devices behind firewall, accessed via jump server
Jump SSH → Device Telnet SFTP Legacy devices behind firewall
Jump Telnet → Device SSH/Telnet SFTP Complex network topologies with legacy jump hosts

9.4 Protocol Selection Logic

PulsarNet automatically selects the most appropriate backup protocol based on device connection type:

  • Direct SSH connections use SCP for secure file transfer
  • Direct Telnet connections use TFTP for compatibility
  • Jump host configurations use SFTP which supports proxying through intermediary hosts

9.5 Enterprise Environment Considerations

PulsarNet's backup implementation accounts for common enterprise constraints:

  • Firewall Traversal: Only the PulsarNet application needs outbound access to jump hosts and storage servers
  • Network Segmentation: Works in environments where network devices are in isolated segments
  • Minimal Access Requirements: Target devices only need to accept connections from jump hosts
  • Security: No outbound connections from sensitive network devices, no credentials for storage servers needed on network devices
